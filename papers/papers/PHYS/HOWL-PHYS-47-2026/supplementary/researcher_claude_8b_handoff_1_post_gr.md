## 2.2 Value Node Format

```json
{
    "key": "mass_z_boson_v0",
    "canonical": "mass_z_boson",
    "version": 0,
    "node_type": "value",
    "topic": "mass",
    "term": "z_boson",
    "level": 2,
    "value": {"_type": "Fraction", "num": "911876", "den": "10"},
    "value_type": "exact_fraction",
    "unit": "MeV",
    "source": "PDG 2024.",
    "tags": ["electroweak", "measured"]
}
```

Values files use `{"nodes": [...]}` wrapper. Value types: `exact_fraction` (Fraction dict with `_type`, `num`, `den`), `exact_integer` (bare int), `approximate` (decimal string, NO scientific notation). Levels: 0=geometry, 1=group theory, 2=measured, 3=derived.

## 2.3 Experiment JSON Format

```json
{
    "key": "experiment_name_v0",
    "canonical": "experiment_name",
    "version": 0,
    "node_type": "experiment",
    "description": "...",
    "purpose": "program_name_v0",
    "experiment_mode": "standard",
    "dependencies": {
        "values": {"canonical_name": 0},
        "derivations": {"canonical_name": 0},
        "connections": {}
    },
    "execution_plan": ["derivation_name_v0"],
    "connections": [],
    "expected_outputs": ["result_something_v0"],
    "comparisons": [...],
    "diagrams": [...]
}
```

Match modes: `miss_pct` (needs `expected`, always INFO), `digits` (needs `expected` + `digits`), `range` (needs `lo`/`hi`), `exact` (needs `expected` Fraction or int), `bool` (needs `expected` true/false).

**CRITICAL:** Every `output_key` in comparisons must EXACTLY match a key in the derivation's return `outputs` dict. Mismatch → SKIP with "output not found in pool". This has been the #1 failure mode across all sessions.

**CRITICAL:** The `bool` match mode has a format issue — derivations output Python string `"True"` but the runner may expect boolean `true`. Watch for FAILs that say `expected True got True`.

## 2.4 Derivation Function Contract

```python
def my_derivation_v0(value_dicts):
    vm = _value_map(value_dicts)
    old_dps = mp.dps
    mp.dps = 50

    x = _f2m(_frac(vm, "some_exact_fraction_v0"))
    y = _mpf_val(vm, "some_approximate_v0")
    z = mpf(str(_get(vm, "some_previous_output_v0")))

    result = x * y

    mp.dps = old_dps
    return {
        "key": "my_derivation_v0",
        "outputs": {
            "result_something_v0": _approx(result),
            "result_miss_pct_v0": _approx(miss),
        },
        "notes": "summary"
    }
```

Reader functions: `_frac(vm, key)` → Fraction, `_f2m(frac)` → mpf, `_mpf_val(vm, key)` → mpf from approximate, `_get(vm, key)` → raw value, `_approx(mpf)` → 15-digit string.

**CRITICAL:** Some pool values are stored as strings, some as Fractions, some as ints. If you get `"astro_gravitational_constant_v0 is not Fraction: <class 'str'>"`, the value is approximate — use `_mpf_val` not `_frac`. Check the value_type in the values file before choosing the reader.
