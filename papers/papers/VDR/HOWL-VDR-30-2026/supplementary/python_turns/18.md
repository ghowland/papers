## What is VDR?

A VDR object is an ordered triple `[V, D, R]`:

- **V** — integer numerator (the settled value in the current denominator frame)
- **D** — nonzero integer denominator (the frame)
- **R** — remainder (exact unresolved structure — atomic integer, composite with child VDRs, or functional callable)

When R = 0, the object is **closed** and behaves as the rational V/D. When R ≠ 0, the object is **active** and carries exact structure that the denominator frame couldn't absorb.

The key rule: **D never explodes.** Multiplication uses `divmod` — the quotient stays in V, the overflow goes to R. D stays fixed. Zero loss.

## Why VDR?

Floating-point arithmetic silently discards information at every operation. Over long chains — video generation, Kalman filters, financial risk aggregation, molecular dynamics — these errors compound.

VDR eliminates per-step arithmetic error entirely:

| Operation chain | Float64 error | VDR error |
|---|---|---|
| 200-op return to origin | ~2.78e-16 | **0** |
| Hilbert 5×5 inverse residual | ~1e-9 | **0** |
| 36,000-step diffusion (30s video) | ~1e-11 | **< 1e-50** |
| IIR filter (1/√2)^20 | ~1e-16 | **0** (exact 1/1024) |
| Spin rotation ×4 = I | ~1e-15 per entry | **0** |

Validated across **921 tests in 38 domains** with **zero VDR computation errors**.

## Q335 Basis

The configurable D-frame. Default is Q335 (D = 2^335), proven across ~1000 tests.

```python
from vdr.basis import set_default, to_qbasis, qb_mul

set_default(bits=335)             # Q335 default
a = to_qbasis(3.14159)           # project onto 2^335 grid
b = to_qbasis(2.71828)
c = qb_mul(a, b)                 # D stays 2^335, overflow in R
```

Any D is valid. Q335 is the default, not the definition.

## Design Principles

1. **Remainder is first-class** — R carries exact unresolved structure that scalar systems discard
2. **D never explodes** — multiplication uses divmod, overflow goes to R
3. **No required dependencies** — Python 3.8+ stdlib only (mpmath optional for decimal export)
4. **The code is the specification** — every claim verified by executable tests
5. **Honest boundaries** — chaotic dynamics have exponential cost (information-theoretic, not VDR defect); active division by active objects loses divisor structure (v1 compromise)

## Validation

921 tests across 38 mathematical and computational domains. Zero VDR computation errors. All 18 test failures across the development history traced to test-design errors, never to VDR arithmetic.
