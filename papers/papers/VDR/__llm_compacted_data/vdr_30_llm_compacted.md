# principles(id|principle|rationale)
P1|VDR triple [V,D,R] represents exact rational (V+R)/D|R is exact residual within D frame, not error; closed=R is 0, active=R≠0
P2|divmod splits results: Q=product>>bits, S=product&mask|when D is power-of-two, integer division is shift+mask — cheapest operations on any processor
P3|V+R always equals exact integer numerator|no operation discards information; chain length irrelevant to error
P4|D is power-of-two matching hardware register widths|D=2^8 weights, D=2^16 activations, D=2^64 gradients; exactness holds at any D
P5|884 tests across 37 domains, zero VDR computation errors|14 test failures all traced to test-design errors, not arithmetic
P6|three independent cost axes multiply: per-token speedup × token reduction × linear scaling|compound effect reaches 20-70× for structured workloads

# float_wall(id|problem|mechanism|why_unfixable|vdr_resolution)
FW_1|non-determinism|float add not associative; GPU thread scheduling non-deterministic → operation order varies → results change|mathematical property of representation; no engineering makes float add associative|integer add is associative; bit-identical results across runs/platforms/hardware
FW_2|softmax ≠ 1|output sums to approximately 1; systematic probability mass error; wrong token selection at boundaries|intrinsic to float representation of transcendentals|VDR softmax sums to exactly 1; exact rational surrogate validated through training
FW_3|attention degrades with context|longer sequences → smaller individual weights → meaningful differences shrink below float resolution|float resolution fixed while dynamic range of meaningful differences shrinks|exact arithmetic preserves all attention weight distinctions at any context length
FW_4|chain drift|each operation rounds; errors compound; float64 drift ~2.6×10⁻⁷ per element at 25.9M steps|rounding intrinsic to representation; correction passes introduce discontinuities|zero drift at any chain length by construction
FW_5|special values (NaN, Inf, denormal, -0)|NaN propagates silently corrupting results; denormals cause unpredictable perf; defensive code everywhere|representation can produce values that break computation|integers have no NaN/Inf/denormal/-0; overflow preventable at design time
FW_6|diminishing returns from scale|larger models don't make float associative; more data doesn't make softmax exact; longer contexts don't reduce drift|problems are properties of substrate, not model|VDR eliminates substrate limitations; model improvements apply to clean foundation

# decode_legend
# VDR: [V, D, R] triple; (V+R)/D = exact rational; D always power-of-two for ML
# basis shorthand: Q8=D=2^8, Q16=D=2^16, Q64=D=2^64, Q335=D=2^335
# closed: R=0; active: R≠0
# divmod: integer division + modulo; reduces to shift+mask when D is power-of-two
# SFU: Special Function Unit — GPU hardware for transcendentals, 32 ops/SM/cycle on H100, 1/16 tensor core rate
# Barrett reduction: integer division via precomputed multiplicative inverse (multiply + shift)
# ULP: unit in last place — least significant bit magnitude of float mantissa
# throughput units: ops/SM/cycle (GPU), ms (latency), tok/s (inference throughput)
# utilization: fraction of peak hardware throughput achieved by kernel
# TOPS: tera integer operations/second; TFLOPS: tera float operations/second
# KB: knowledge base — scoped storage at integer addresses
# builtin: typed deterministic primitive invoked by LLM (~8 tokens per invocation)
# grammar: structural template providing fixed output tokens with typed content slots
# IOSE: Input, Output, Side Effects, Properties — declaration model for every VDR component
# execution levels: L1 (full LLM), L2 (LLM+Prolog), L3 (pure Prolog)
# token reduction %: fraction of conventional tokens eliminated by deterministic components
# cost factor: conventional_cost / vdr_cost
# scaling: conventional=quadratic T×N(N+1)/2; VDR=linear N×C
# rel_types: defines|enables|validates|composes|bottlenecks|eliminates_bottleneck|replaced_by|produces|prevented_by|mitigated_by|motivates|eliminated_by|requires|implements|captures|amplifies|bounds|matures_to
